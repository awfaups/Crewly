# Crewly 技能市场安装语义修正 - 目标与风险对齐

## 当前问题

上一版把“技能安装入口”实现为“把技能分配给 AI 成员”。用户澄清后，正确语义应为：从外部 Skill 市场把 skill 安装到 Crewly 的技能市场。

## 正确产品边界

- Skill 市场：外部/平台侧可安装 skill 来源。
- Crewly 技能市场：当前 Crewly 工作区已安装 skill 的集合。
- AI 成员独立技能：AI 成员只能从 Crewly 已安装 skill 中独立启用和调用。

## 风险

- 需要兼容旧 localStorage 数据。
- 不能破坏 AI 成员已有独立记忆、模型配置和技能调用审批闭环。
- 需要避免 UI 再次把“安装到 Crewly”和“成员启用”混为一谈。

## 回滚预期

本次改动集中在前端状态和技能市场 UI，回滚提交即可恢复上一版行为。
