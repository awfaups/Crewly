# Crewly 后端基础设施 - 需求对齐

## 背景

当前 Crewly 是前端静态 demo，数据仍来自 mock 与 `localStorage`。用户确认开始后端建设。

## 本轮目标

- 建立 Fastify 后端入口。
- 建立 PostgreSQL/Drizzle 数据模型。
- 提供健康检查和核心资源读取 API 骨架。
- 保持前端现有 demo 可用，不立即切换远程数据源。

## 非目标

- 不接真实登录。
- 不实现写入 API。
- 不接真实 AI 模型调用。
- 不部署独立后端服务。

## 约束

- 密钥只通过环境变量读取。
- 不提交真实 `DATABASE_URL`。
- 后端启动时没有数据库也应有清晰降级响应。
