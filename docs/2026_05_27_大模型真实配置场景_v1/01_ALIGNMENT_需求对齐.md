# 大模型真实配置场景 - 需求对齐

## 背景

当前 AI 成员已支持填写模型提供商、模型名称和接入说明，但这更像演示备注，不足以表达真实大模型接入配置。

## 目标

- 将 AI 成员的大模型配置升级为真实接入场景常见字段。
- 支持 Base URL、认证方式、密钥引用名、环境、组织/项目、参数和能力开关。
- 保持安全边界：不保存真实 API Key。
- 兼容已有本地数据中的旧模型配置。

## 范围

- 前端配置建模和展示。
- localStorage 本地演示持久化。
- 不发起真实模型请求。
- 不校验 API Key 或模型可用性。

## 关键约束

- API Key 只能以引用名形式保存，例如 `OPENAI_API_KEY`。
- 不把密钥明文写入浏览器本地存储。
- 配置字段要贴近后续服务端接入，不只是一段说明文本。
