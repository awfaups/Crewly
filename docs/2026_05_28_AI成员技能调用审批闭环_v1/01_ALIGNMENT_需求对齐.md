# AI 成员技能调用审批闭环 - 需求对齐

## 背景

Crewly 已支持 AI 成员技能库、成员级独立安装和调用策略配置。当前缺少“从任务上下文触发技能调用，并把结果写回 Agent Session 和审批流”的演示闭环。

## 目标

- 在右侧 AI 成员详情中展示当前成员可调用技能。
- 点击技能后向当前任务的 Agent Session 追加技能调用事件。
- 低风险技能直接追加模拟结果。
- 高风险技能或启用外部动作审批时创建审批请求。
- 审批通过或拒绝后继续回写 Agent Session。
- 保持本地 demo，不执行真实外部技能调用。

## 非目标

- 不调用真实模型。
- 不访问真实外部网页、仓库或文件系统。
- 不实现技能运行沙箱。
- 不新增后端 API。

## 假设

- 技能风险等级使用现有 `SkillCatalogItem.riskLevel`。
- AI 成员的 `requireApprovalForExternalActions` 控制是否需要审批。
- 审批卡继续复用现有 `Approval` 数据结构。
