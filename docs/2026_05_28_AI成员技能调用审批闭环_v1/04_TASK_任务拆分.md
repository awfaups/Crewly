# AI 成员技能调用审批闭环 - 任务拆分

## T1 新增调用入口

- 输入：右侧 AI 成员详情。
- 输出：展示当前成员已安装技能按钮。

## T2 实现调用状态更新

- 输入：当前 member、task、session、skill。
- 输出：追加 Agent Session 事件。

## T3 接入审批

- 输入：技能风险和成员审批策略。
- 输出：创建审批请求并展示审批卡。

## T4 持久化兼容

- 输入：localStorage workspace state。
- 输出：刷新后事件和审批保留。

## T5 验证

- 运行 lint、build。
- 浏览器验证低风险调用、高风险审批、刷新持久化。
