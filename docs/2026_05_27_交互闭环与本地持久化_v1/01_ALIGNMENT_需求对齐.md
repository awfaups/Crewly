# 交互闭环与本地持久化 - 需求对齐

## 背景

Crewly 当前已完成中文三栏工作台 MVP，但核心数据仍是静态 mock：

- 任务状态不可流转。
- 审批结果只改变审批卡自身状态。
- 消息输入框不可真正新增消息。
- 页面刷新后所有操作丢失。

## 本轮目标

把 Crewly 从静态原型升级为可操作 demo：

- 任务卡支持状态流转。
- 审批通过或拒绝后同步影响任务和 Agent Session。
- 消息输入框支持向当前频道新增消息。
- 使用 `localStorage` 保存任务、审批、Session 和消息状态。
- 提供一键重置演示数据能力。

## 范围

本轮只做浏览器端交互闭环，不接后端。

纳入：

- 前端状态管理。
- localStorage 持久化。
- 中文界面文案。
- README 和验收文档同步。

不纳入：

- 登录和用户体系。
- 后端 API。
- 数据库。
- WebSocket。
- 真实 Agent runtime。
- 多人协作冲突处理。

## 验收口径

- 切换任务状态后，任务看板和右侧详情同步更新。
- 审批通过后，相关任务进入待验收或已完成的合理状态，Session 状态同步变化。
- 审批拒绝后，相关任务回到进行中，Session 保持运行中或等待修正。
- 新增消息立即出现在当前频道。
- 刷新页面后仍保留操作结果。
- 点击重置后恢复初始 mock 数据。
